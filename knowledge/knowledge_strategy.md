# 知识结构与检索策略选型

## 1. 背景

小安的知识库目标不是做普通百科问答，而是支持家暴受害者在高压、低安全感、信息不完整的情况下，获得：

- 情绪与认知承接；
- 低负担行动建议；
- 本土法律与实务依据；
- 必要时可逐层展开的法律/案例/操作细节。

目前可用资料包括：《实用手册》、国家反家暴法、司法解释、地方实施细则、部门规范、NGO/社工经验、案例材料等。这些资料来源多、颗粒度不一、语体差异大，不能简单等同于“上传文档即可生成好回答”。

直接丢法条给大模型（传统 RAG），等于在要求大模型在短短 2 秒钟的生成时间里，同时完成三件事：

1. 像法官一样精准检索法条全文。
2. 像资深律师一样把法条翻译成可执行的操作（SOP）。
3. 像心理咨询师一样把生硬的操作用极其温和、安抚的语气说出来。

目前的通用大模型**根本做不到**在这三者之间完美平衡，它必然会“顾此失彼”——要么丢给你一堆冰冷的法条（用户看不懂也做不到），要么为了共情而编造一些看似合理但实操中极其危险的假建议（幻觉）。

## 2. 方案比较

### 方案 A：基于向量相似度的意图到标准作业映射 (Intent-to-SOP Semantic Routing)

- **机制：** 将高频问题（如“怎么报警”）向量化。用户输入匹配命中后，直接提取预设的纯文本标准答案（SOP），大模型仅做简单的语气润色。
- **优势：** * 检索耗时极低（O(1) 级别，通常 < 50ms）。
- **劣势（弃用原因）：**
  - **交互呈机械触发式**：本质上是一个“高级智能客服”，缺乏连贯的上下文共情和多轮对话推演能力。
- 结论：只能作为系统崩溃兜底或极度高危（如正在遭遇暴力求救）的紧急拦截网，不能作为核心体验。

### 方案 B：基于暴力循环的时间轴架构 (User Journey Timeline)

- \*\*机制：\*\*将知识按受害者生命周期（潜伏期 ➔ 爆发期 ➔ 取证期 ➔ 诉讼期 ➔ 恢复期）进行物理隔离。先用小模型判断用户阶段，只在对应阶段的知识池内检索。
- **优势：** * 宏观防错：从根本上防止给“正在挨打”的用户背诵“离婚法条”这种荒谬错误。
- **劣势（弃用原因）：**
  - **状态跳跃处理复杂：** 真实用户的状态通常是非线性的，状态机的频繁跳转和回溯会显著增加中间件的维护成本。
- 结论： 更适合作为状态标签。

### 方案 C：场景胶囊架构 / R-A-G 胶囊

**Recognize–Act–Ground Scenario Capsules**

**机制：**
以“具体用户痛点”为基本知识单元，而不是按原始文档、法条章节或暴力阶段存储知识。每个场景胶囊围绕一个高频问题构建，例如“警察说这是家务事不管怎么办”，内部拆成三个回答槽位：

- **Recognize /认知**：帮助用户理解处境，降低自责和自我怀疑
- **Act / 行动**：给出低负担、相对安全的下一步
- **Ground / 依据**：提供法律、规范、案例与实务依据，并说明边界

命中胶囊后，系统不默认展示全部内容，而是根据用户状态和追问意图选择性展开。

**优势：**

- **兼顾共情与行动**：不是简单返回 SOP，而是让大模型基于结构化内容自然回应，保留多轮对话能力。
- **能消化复杂资料**：可以把法律条文、地方细则、部门规范、社工经验、案例逻辑转译成用户可理解、可行动的支持。
- **更能形成壁垒**：真正沉淀的是“本土反家暴实务经验如何转化为对话支持”，而不是单纯资料检索。

**劣势 / 成本：**

- **初期人工成本高**：不能直接上传 PDF 完成，需要人工或 AI 辅助拆解、归纳、重组。
- **需要专家审核**：因为胶囊会影响行动建议，核心场景必须经过法律/社工/一线经验审核。
- **覆盖不足时需要兜底**：未命中胶囊时仍需回退到普通 Main Agent 或基础 RAG。

## 3.如何落实方案C

**策略一：二八定律（不要一口气做完）** 在 0 到 1 阶段，千万别试图穷尽所有的家暴场景。 真实世界中，80% 的受害者面临的都是那集中的 20% 的困境（比如：怎么留证据、警察不管怎么办、想离婚怕抢不到孩子）。 你们只需要先集中精力人工打磨这 **Top 10 的高频场景胶囊**。

如果用户问到了冷门场景（没有命中胶囊），系统就退回到你们设计的“普通对话弱控”模式，让 Main Support Agent 纯靠共情去安抚，或者坦诚地说：“这个情况比较复杂，我帮你连接专业热线好吗？”

**策略二：让用户的真实提问指导你们的下一步** 产品上线后，你们去后台看日志（这也是为什么要自己写后端的一大好处）。看看用户实际上在问什么，哪些问题小安接不住。下周的开发迭代，你们就专门针对这几个真实发生的高频痛点，去人工补充新的“场景胶囊”。这叫**数据飞轮驱动的内容建设**。

总结来说：这是一件先苦后甜的事。前期花几十个小时梳理出这些高价值的胶囊，换来的是上线后极高的安全性、极低的延迟，以及别人无法抄袭的核心竞争力。

### 💡 核心梳理心法：倒推法

很多人梳理知识库失败，是因为他们拿着《反家暴法》第一条往下读，边读边想“这条怎么拆”。这是错的。

**正确的梳理方法是“倒推法”：**

1. **不要看书，先看人。** 闭上眼睛想，或者看真实的社工接案记录，写下 30 个受害者最常问的、最绝望的“具体问题”（这就是 Trigger）。比如：“警察来了但在门外抽烟不管”。
2. **针对每一个问题，去材料里“寻宝”。**
   1. 为了解决“警察不管”，我需要让她去要警号（Act）。
   2. 为了让她有底气要警号，我去翻《公安机关执法细则》或省条例，找到“必须出警记录”的条文（**Ground**）。
   3. 最后，我给她加一句认知教育：“你要求他办事是合理的”（**Recognize**）。
3. **一个痛点 + 一套 r-a-g = 一个胶囊组装完毕。**
