# 學術論文層設計規範與操作手冊

本文件規定小安知識庫中**學術論文層**的設計原則、節點格式、抽取規則、NotebookLM 查詢模板，以及新論文的入庫流程。

---

## 一、為什麼引入學術論文層

現有 Tier 1–4 來源（法條、司法解釋、典型案例、地方條例）回答的是「法律說什麼」。學術論文層回答的是「法律為什麼這樣說、邊界在哪裡、為何在實踐中失靈、背後的理論基礎是什麼」。

學術論文服務於五種不同的**認識論功能（Epistemic Function）**：

| 代碼 | 功能 | 作用 |
|------|------|------|
| F1 | **定義縱深** | 法條邊界在哪裡、模糊地帶如何認定 |
| F2 | **立法溯源** | 為什麼立法者選了這個詞、借鑑了哪些域外制度 |
| F3 | **司法現實** | 法律在實踐中為何失靈、法官實際怎麼推理 |
| F4 | **法哲學基礎** | 支撐法條的底層原則（公私領域、國家保護義務） |
| F5 | **倡導框架** | AI 的隱性認知底色——女性主義視角、國際人權標準 |

**重要**：學術論文**無法律強制力**。AI 引用時必須使用對應語言（見第六節），絕不能用「法律規定」「法院認定」等 Tier 1 用語。

---

## 二、來源類型定義

在現有 `knowledge/AGENTS.md` 的信任分級基礎上，新增第五層：

| Tier | `source_type` | 說明 |
|------|--------------|------|
| 5 | `academic_paper` | 法學期刊論文（核心期刊/CSSCI）、碩博士學位論文、立法諮詢中的學者意見文章。**無法律強制力，僅供詮釋參考。** |

規則：
- `academic_paper` **不能**用來推導法律義務（不能說「依據某論文，警察必須…」）
- `academic_paper` **可以**說明立法意圖、概念邊界、比較法來源、司法行為模式
- 論文觀點與 Tier 1 衝突時，以 Tier 1 為準，但保留論文批評以供 F5 倡導使用
- 2016 年前的論文需特別注意時效（見第五節時效協議）

## 三、新節點類型：`concept-depth`

學術論文節點使用新的 `node_kind: concept-depth`，格式如下：

```yaml
---
type: legal-node
node_kind: concept-depth
title: ""
epistemic_function: F1  # F1/F2/F3/F4/F5 其中之一
academic_binding: none   # 固定值，提醒 AI 此節點無法律強制力
source_refs:
  - "knowledge/source/論文_pdf/論文標題.pdf#章節或頁碼"
temporal_note: ""        # 時效說明，見第五節
updated: YYYY-MM-DD
status: draft
---
```

节点内容结构（固定格式）：


## 四、新邊類型

以下三種邊類型已正式登記於 `AGENTS.md`（schema v0.1）及 `edges.md`（v0.5）：

| 關係 | 含義 |
|------|------|
| `deepens` | concept-depth 節點為現有 definition/condition 節點補充語義深度（主要用於 F1） |
| `traces_legislative_origin_of` | 說明某概念的域外立法來源（主要用於 F2） |
| `contextualizes_reality_of` | judicial-reality 節點解釋某法律機制在實踐中為何失靈（主要用於 F3） |

連接慣例（從 concept-depth 出發）：
```
psychological-violence-concept --deepens--> domestic-violence-definition
coercive-control-comparative --traces_legislative_origin_of--> domestic-violence-definition
judicial-recognition-gap --contextualizes_reality_of--> protection-order-evidence
victim-agency-barriers --contextualizes_reality_of--> judicial-recognition-gap  # Tier 5→Tier 5 允許
```

**錨定規則**：每個 concept-depth 節點至少須有一條邊指向 Tier 1–4 節點，確保學術層錨定在法律樹上、不形成浮島。Tier 5 → Tier 5 邊可以建立，但不能替代 Tier 1–4 錨點。

### 5.1 `psychological-violence-concept` — 精神暴力：定義縱深
**功能**：F1  
**核心問題**：他只是罵我/冷戰/監控我，這算家暴嗎？

**NotebookLM 查詢模板**：
```
1. 學界如何定義「精神暴力」（心理暴力）？核心特徵是什麼？不同學者的定義有何差異？

2. 精神暴力是否必須具備「持續性/經常性」？哪些情況下單次行為也可構成？請引用具體論文。

3. 請列舉學界認為屬於精神暴力的具體行為類型（包括但不限於冷暴力、監控、孤立、網絡手段）。哪些存在認定爭議？

4. 「冷暴力」（不作為的精神暴力）是否應納入家暴範疇？學界主要分歧是什麼？

5. 基層法院和執法人員在認定精神暴力時，與學界定義存在哪些具體落差？請引用實證數據。
```

**連接節點**：`deepens` → `domestic-violence-definition`

### 5.2 `economic-control-concept` — 經濟控制：定義與立法空白
**功能**：F1  
**核心問題**：他控制我所有的錢/不讓我工作，這算家暴嗎？

**NotebookLM 查詢模板**：
```
1. 學界如何定義「經濟控制」（經濟虐待/財產暴力）？核心特徵是什麼？請引用具體論文和定義。

2. 為什麼國家《反家庭暴力法》沒有明文列舉經濟控制？立法機關的說法是什麼？學界如何解讀這個空白？

3. 關於經濟控制是否應獨立成類，學界有哪些主要觀點？「涵蓋說」和「獨立說」各自的論據是什麼？

4. 哪些地方條例已經明文列舉了經濟控制？具體表述是什麼？

5. 學界認為經濟控制的具體行為表現有哪些？（如截留工資、強迫借債、妨礙就業等）
```

**連接節點**：`deepens` → `domestic-violence-definition`

> 參照：部分地方條例已明文列舉經濟控制（見 `local-regulations-comparison`），可作為佐證，但不構成 schema 邊。

```
4. 美國《暴力侵害婦女法》（VAWA）和韓國、南非等國家的家暴定義如何擴展了主體和行為範圍？
```

### 5.6 `international-human-rights-standards` — 國際人權標準：立法溯源與比較

**NotebookLM 查詢模板**：
```

3. 1993年《消除對婦女的暴力宣言》和1995年北京行動綱領對精神暴力、性暴力的規定是什麼？

4. 學界認為中國作為CEDAW締約國，在家暴保護上還有哪些未履行的國際義務？

5. 《兒童權利公約》（CRC）對家庭環境中兒童免受暴力的保護有哪些要求？學界如何評估中國在兒童目睹家暴、兒童受暴保護方面的履約情況？

6. 「兩公約」（ICCPR/ICESCR）在反家暴方面有哪些相關條款？學界如何討論這些義務？
```

**連接節點**：`deepens` → `domestic-violence-definition`；`deepens` → `liability-ladder`

## 六、時效協議

論文觀點可能因立法修改而過時，每條 claim 需標注時效狀態：

| 標記 | 含義 | 範例 |
|------|------|------|
| `[現行有效]` | 在2026年仍然準確 | 精神暴力需「經常性」的立法要求 |
| `[倡議→已實現 YYYY]` | 曾是學界倡議，現已入法 | 2017年倡議經濟控制入法→部分地方條例已實現 |
| `[倡議→待實現]` | 仍是未實現的學界倡議 | 將前任伴侶納入保護範圍 |
| `[時效待核]` | 不確定，需對照現行法核實 | 任何2016年前發表的論文觀點 |

**操作規則**：ingest 新論文時，對所有「學界建議未來如何修法」的陳述自動標注 `[時效待核]`，並在更新節點時對照當前反家暴法及地方條例核實。

## 七、通用抽取規則

### 7.1 應抽取的內容

| 類別 | 說明 | 注意事項 |
|------|------|----------|
| **概念定義與邊界論述** | 學者對某法律概念的定義、邊界討論 | 記錄論文作者和觀點，不做平均 |
| **具體行為列舉** | 論文認為屬於某概念的行為類型 | 注明是「學界主張」不是「法律規定」 |
| **立法意圖直接引用** | 立法機關說法、起草過程中的說明 | 這類引用可信度較高，近似Tier 2 |
| **域外法律制度** | 英國、台灣、EU 等具體制度條款 | 只抽取與中國法有明確關聯的部分（被借鑑或被對比） |
| **學界內部分歧** | 兩種以上觀點的爭議 | 雙方都保留，標明各自支持者 |
| **實證統計數據** | 認定率、樣本數量、地域比較 | 記錄研究年份、樣本說明 |
| **學界批評立法的論點** | 對現行法律的批評性分析 | 標注批評是否已被後續立法回應 |
| **學界倡議的改進方向** | 建議未來如何修法 | 一律標注時效狀態（見第五節） |
| **論文引用的案例** | 論文分析的具體案例 | 標注判決年份；比媒體報道更精確 |
| **國際公約條款引用** | CEDAW、伊斯坦堡公約等具體條款 | 標注公約名稱、條款編號；說明對中國的約束力 |

### 7.2 不應抽取的內容

| 類別 | 原因 |
|------|------|
| 方法論章節、問卷設計 | 無學術觀點，對AI無用 |
| 純文獻綜述段落 | 轉述他人觀點，不是一手論述；原論文已入庫 |
| 與中國法無連接的純域外法介紹 | 若論文只介紹英國法但未連結中國，不抽取 |
| 論文重複其他已入庫論文的觀點 | 節點里第一次出現的論述才記錄；後續只在`source_refs`加引用 |
| 純政策倡議（非法律分析） | 「政府應增加資金投入」等不涉及法律解釋的建議 |
| 社會工作操作流程 | 那是另一類知識，不屬於法學概念層 |

### 7.3 學界分歧的處理規則

- **分歧雙方都保留**，在節點的「主要爭議與分歧」小節中並列呈現
- **標明主流與少數**：若學界有明顯主流立場，說明「多數學者認為…但也有學者（如X）主張…」
- **不自行裁決**：節點不下結論「哪方正確」，AI 在回答時根據問題場景選用
- **與 Tier 1 衝突時**：保留學界觀點，但在節點中注明「現行法律規定…但學界批評認為…」

## 八、AI 引用語言規範

學術節點**無法律強制力**，AI 必須使用以下對應語言，絕不能用 Tier 1 的「依法」「法律規定」「法院認定」。

| 功能 | 允許使用的措辭 | 禁止使用的措辭 |
|------|----------------|----------------|
| F1 定義縱深 | 「學界通說認為」「有學者主張」「法學界討論認為」「學術上一般認為」 | 「法律規定」「依法」「認定標準為」 |
| F2 立法溯源 | 「立法過程中」「起草階段的討論」「參照了…的立法模式」「借鑑自…」 | 「現行法律要求」 |
| F3 司法現實 | 「實證研究顯示」「研究數據表明」「根據對X份判決的分析」 | 「法院認定」「司法實踐規定」 |
| F4 法哲學 | 「從法理角度」「法學理論認為」「憲法學者指出」 | 任何暗示這是現行規定的表述 |
| F5 倡導框架 | **不直接引用**，作為 AI 措辭傾向的隱性底色 | 不得以「有學者認為女性應該…」方式說教 |

**F5 特別規則**：性別理論和倡導框架**不透過 ground resolver 或膠囊 locator 注入**，而是透過 system prompt AI 立場聲明全局生效（見 `tech/chatflow/design-doc.md` §11）。立場聲明的具體行為規範：
- 不把受害者行為歸咎於個人（如「你為什麼不早離開」之類的隱含責備）
- 對「這是家務事」的陳述保持敏感、不認可
- 解釋受害者困境時優先使用結構性語言

## 九、新論文入庫流程

每次新增一篇或一批論文時，按以下步驟操作：

### 9.1 準備工作
1. 將 PDF 放入 `knowledge/source/論文_pdf/`
2. 在 `knowledge/wiki/source-registry.md` 新增登記：
   ```
   | 論文標題 | academic_paper | 5 | pending |
   ```

### 9.2 上傳到 NotebookLM
```bash
notebooklm source add "knowledge/source/論文_pdf/論文名.pdf" --json
```
Notebook ID：`b14e6843-9ddf-486a-9bf4-a8345687423b`

等待 source status 變為 `ready`。

### 9.3 使用 NotebookLM 查詢

針對新論文，逐一運行相關節點的查詢模板（見第五節）。判斷每個節點是否需要更新：
- 若論文提供了**新定義、新數據、新案例、新爭議觀點** → 更新對應節點
- 若論文只是重複現有節點已記錄的觀點 → 只在節點 `source_refs` 中新增引用，不改正文

### 9.4 更新 wiki 節點

對需要更新的 concept-depth 節點：
1. 在對應小節加入新內容，附 `source_ref`：論文文件名 + 頁碼/章節
2. 新的學界倡議觀點標注時效狀態（見第五節）
3. 若發現學界分歧，在「主要爭議」小節補充

### 9.5 更新登記和圖谱
1. 更新 `source-registry.md`：將狀態改為 `done`，在備注欄標明貢獻到哪些節點
2. 若創建了新的 concept-depth 節點：在 `edges.md` 新增邊
3. 更新 `knowledge/index.md`
4. 在 `knowledge/log.md` 追加記錄：
   ```
   ## [YYYY-MM-DD] ingest | 論文標題 | academic_paper
   貢獻節點：psychological-violence-concept（新增冷暴力分歧觀點）
   ```

## 十、與現有法律機制樹的整合規則

### 10.1 文件位置
- concept-depth 節點放在 `knowledge/wiki/nodes/` 與現有法律原子節點同目錄
- 在節點文件名前**不加**前綴，使用語義化命名：`psychological-violence-concept.md`

### 10.2 Obsidian 圖谱規則
- concept-depth 節點通過 `## 機制關係` 中的 wikilink 連接到現有節點
- 範例：
  ```markdown
  ## 機制關係
  - `deepens` → [[domestic-violence-definition]]
  - `contextualizes_reality_of` → [[protection-order-evidence]]
  ```
- edges.md 中對應補充邊記錄（用 inline code，不用 wikilink）

### 10.3 index.md 更新
academic 節點在 index.md 中單獨分組，標注「學術論文層（非約束性）」：
```
## 學術論文層節點（Tier 5，無法律強制力）
- [[psychological-violence-concept]] — 精神暴力定義縱深、司法認定困境
- [[economic-control-concept]] — 經濟控制定義、立法空白、學界分歧
...
```

### 10.4 優先順序規則
AI 在回答時：
1. 先找 Tier 1 法律節點確定**法律規定是什麼**
2. 再找 F1/F2/F3/F4 concept-depth 節點補充**定義邊界、司法現實、立法背景**
3. F5 節點（`gender-power-analysis`）的框架透過**系統層 AI 立場聲明**體現於所有對話，不作引用來源、不進膠囊 ground

### 10.5 膠囊 locator 整合規則

F1–F4 的 concept-depth 節點**可以**出現在膠囊的 `basis` locator 中，但受以下限制：

- **只能放在 `practice_basis`**，不能放在 `legal_basis`
- `legal_basis` 只引用 Tier 1–4 節點（法條、司法解釋、地方法規）
- ground resolver 回傳 concept-depth 節點內容時，`⚠️ 學術來源` 警告框會一同回傳，LLM 須使用對應語言（見第八節）

**F5 例外**：`gender-power-analysis` **不得**出現在膠囊 `basis` locator 中。其框架透過 system prompt 的 AI 立場聲明全局生效（見 `tech/chatflow/design-doc.md` §11），結構性語言需要特別強調的膠囊可在 `ai_stance` 欄位聲明（非 locator）。

locator 寫法範例（F1–F4）：
```yaml
practice_basis:
  - "學界通說認為精神暴力無需「持續性」要件（參見學術分析）"
basis:
  - locator: "node:psychological-violence-concept"
    note: "精神暴力學界定義縱深（Tier 5，學術來源，非法律強制力）"
```
