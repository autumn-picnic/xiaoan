# triggers
- "我该怎么准备逃跑/离开"
- "我想保护自己（和孩子）"
- "万一他/她打我怎么办"
- "离开之前要做什么"
- "我想提前做些准备"
- "他/她要是找到我怎么办"

# applicability

use_when:
- 用户想了解如何制定安全计划（无论是否决定离开）
- 用户想在留下的同时降低风险

do_not_use_when:
- 用户正在遭遇暴力或当前不安全 → 转接【即时危机】
- 用户明确拒绝讨论安全话题

override:
- 用户说"不想离开"但想了解保护自己 → 本胶囊仍适用，侧重"留下时更安全"

# recognize

## safety_first_notice（每次对话最先传达）
告知用户：如果使用对方可能查看的设备，请用无痕模式或阅后清除记录。

## 核心认知框架
1. 安全计划≠一定要离开。即使留下，计划也能降低风险。
2. ⚠️ 离开的过程是最危险的时刻——不鼓励在争吵升级时仓促逃离，应在平静时提前准备。
3. 不施压离开。用户的节奏优先。
4. 每个人的处境不同，计划必须个人化。

## 评估问题（用于个人化建议）

| # | 问题 | 分支 |
|---|------|------|
| 1 | 同住人员？（孩子/老人） | 有孩子 → 触发儿童相关指引 |
| 2 | 有安全落脚点吗？ | 无 → 提供庇护资源 |
| 3 | 能秘密存钱吗？ | 不能 → 替代经济建议 |
| 4 | 手机是否被追踪/监控？ | 是 → 强化通讯安全 |
| 5 | 对方有过身体暴力吗？ | 有 → 提高紧迫性 |
| 6 | 想准备离开，还是留下但更安全？ | → 决定内容侧重 |

# act

act.instruction: 根据评估结果，从以下模块中选择相关内容生成个人化安全计划。模型可自行展开具体建议，以下为每个模块的**关键点和易遗漏点**。

## 模块清单

**1. 人身安全（所有用户）**
- 关键点：熟悉出口路线、确定安全房间（有出口、无利器）、与邻居/亲友约定暗号
- 易遗漏：不要试图"赢"争吵，优先脱离

**2. 联络网络**
- 关键点：至少一位知情的信任人、紧急暗语、重要号码要记住而非仅存手机
- 易遗漏：联系人号码存为不引人怀疑的名字

**3. 经济准备**
- 关键点：小额现金积累、独立账户
- 条件分支（无法存钱）：信任人处预存、了解紧急救助金渠道

**4. 通讯与数字安全**
- 关键点：检查定位/云账号共享、关闭社交媒体签到
- 条件分支（被监控）：使用备用设备、公用电话；手机留在日常位置
- 易遗漏：云账号（iCloud/华为云/小米云）是常见监控渠道

**5. 安全包**
- 关键点：模型可自行列出物品清单
- 易遗漏：存放位置应在家外（亲友处/工作场所），不是家中

**6. 离开时机（计划离开的用户）**
- ⚠️ 核心原则：在平静时行动，不在冲突中仓促逃离
- 易遗漏：离开后需立即处理——更换号码、关闭共享定位、通知学校/单位

**7. 有孩子时的额外准备**（评估#1触发）
- 关键点：孩子证件、用适龄方式沟通、了解抚养权法律
- 易遗漏：通知学校限制对方接送权限

**8. 留下时的安全策略**（评估#6触发）
- 关键点：识别暴力升级信号、保持安全包和联络网就绪
- 易遗漏：定期重新评估——如暴力升级，重新考虑离开

# safety_note（每次对话结尾提醒）
- 查阅后清除记录或用无痕模式
- 不要向对方透露你在做准备
- 计划不要存在对方可能看到的地方

# ground

limits:
- 不能保证绝对安全，实际需灵活应对
- 不替代专业社工面对面评估
- 不构成法律建议

hotline:
- 全国妇联维权热线：12338
- 报警：110
