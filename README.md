# 一句话定义小安

> 一句话定义小安：不是更会聊天的大模型，而是一个安全、有本土实务知识、能把复杂现实拆成用户可承受下一步的第一响应系统。小安真正的差异化在于本土实务、法律执行现实、一线社工经验和安全边界。

## 1. Problem｜问题

家暴受害者在最需要帮助的时候，往往没有及时、可及、低门槛的支持入口。

专业资源如社工、律师、庇护资源有限，触达成本高；而在正式求助前，用户往往已经经历了长时间的情绪反复、犹豫、恐惧与信息不对称。

现有通用大模型虽然能提供一定情感陪伴，但在家暴场景中仍有明显不足：

- 缺乏中国本土法律与现实执行经验；
- 难以理解警察推诿、基层和稀泥、资源稀缺等现实情境；
- 容易给出“报警”“离开”“离婚”等过度简化甚至有风险的建议；
- 对残障、老年、少数性别、低资源女性等边缘人群覆盖不足。

结果是：用户要么无法获得支持，要么得到看似正确、实际不适用甚至可能增加风险的建议。

## 2. Insight｜关键洞察

我们在专家工作坊和产品测试中确认了几个重要判断：

第一，用户在求助专业人士之前，已经在和 AI 反复对话。AI 在情绪承接、反复陪伴、低门槛可及性上有天然优势。

第二，通用大模型在基础共情上已经很强。小安的价值不应该是重新发明“情感陪伴”，而是把它和本土反家暴实践知识、安全判断、行动路径结合起来。

第三，当前产品测试中，部分结果不如 generic LLM，说明问题不只是模型能力，而是我们早期 flow 和知识库设计限制了模型发挥：过度分类、过早路由、硬性 RAG 介入，会让对话变得死板，失去真实支持者的连续理解能力。

第四，真正缺失的是一套能把一线经验、法律材料、部门规范、案例逻辑转化为可检索、可展开、可审计的知识结构。

小安真正要解决的是：

> **如何把复杂的法律现实、社工经验和制度执行差异，转译成用户当下能承受、能理解、能采取的一小步。**

因此，AI 的角色不是替代社工或律师，而是成为一个安全意识强的“第一响应层”：更早接住用户，更低负担地提供支持，并在复杂情境中引导用户连接专业资源。

## 3. Solution｜解决方案

**小安（XiaoAn）** 

核心能力包括：

### 风险识别

识别用户是否处于即时危险、刚经历暴力、或需要优先进行安全确认。

### 情感支持

提供稳定、无评判、创伤知情的回应，让用户感到被接住，而不是被审问或被指挥。

### 行动引导

根据用户状态，提供少量、低负担、可执行的下一步，例如安全确认、报警沟通、证据保存、求助路径。

### 本土知识补充

引入中国反家暴法律、部门实务、警方应对、社工经验和类案逻辑，减少通用大模型的空泛回答和幻觉风险。

## 4.  Technical Approach｜技术路线

我们采用：**安全强控 + 普通对话弱控 + 分层知识增强**

### 4.1 安全强控：Global SOP

高风险场景不交给普通 RAG 或自由对话处理。

如果用户表达：

- 正在遭遇暴力；
- 刚刚被打；
- 施暴者在附近；
- 手机可能被监控；
- 当前安全状态不明；

系统优先进入 **Crisis SOP**：

- 不展开长篇法律知识；
- 不给复杂行动清单；
- 不鼓励对抗；
- 只确认安全并给 1–2 个立即可做的小步骤。

另有 **Listen SOP**：当用户只是想被听见时，不主动给建议、不追问暴力细节、不催促报警/离开/离婚，只在用户出现转向信号时轻轻桥接下一步。

### 4.2 普通对话弱控：Main Support Agent

非高危场景不使用复杂多分支 flow，也不硬切成“法律 / 心理 / 行动”多个 agent。

统一由 **Main Support Agent** 承接：

- 保留大模型自然对话能力；
- 能处理混合需求；
- 不强行塞知识库；
- 只在相关时使用场景胶囊；
- 输出前经过 Output Guard，避免命令式、承诺式、责备式或危险建议。

### 4.3 场景胶囊：P0 核心知识单元

P0 不建设完整法律机制树、独立知识卡系统或独立可展开地图系统。
这些能力暂时降级为胶囊字段，避免过度工程化。

每个 [[R-A-G 场景胶囊](/lNlObSITSB-c4zhOK11Mbg)] 围绕一个具体用户痛点构建，例如：

- 警察说这是家务事 / 不处理；
- 报警后如何留下记录；
- 家暴证据怎么保存。

![image](https://hackmd.io/_uploads/B1VNsa-eGe.png)

### 4.4 SOP 与胶囊的关系

SOP 和场景胶囊不完全分开，也不完全合并。

![image](https://hackmd.io/_uploads/BkvKiTWlzl.png)

### 4.5 可展开地图：P0 作为胶囊字段

P0 不做独立知识地图系统。

可展开地图先作为每个胶囊里的 `related_topics` 字段存在，用于告诉用户：

> 这个问题之后，还可以继续了解哪些相关方向？

例如在“警察说这是家务事”胶囊中：

```
related_topics:
- 报警后如何留下记录
- 告诫书是什么
- 警察只调解不记录怎么办
- 家暴证据怎么保存
- 报警记录以后有没有用
```

这能提供渐进式探索，但不会把 P0 做成复杂知识平台。

## 5. Differentiation｜差异化

小安的壁垒不在于“比大模型更会聊天”，而在于：

- **本土实务知识**：理解中国反家暴一线场景，而不是只输出通用建议；
- **法律执行现实**：知道法律文本和现实执行之间存在距离；
- **一线社工经验**：能把隐性经验转译成用户能用的话术和步骤；
- **安全边界**：高风险场景优先保护用户，而不是追求回答完整；
- **低负担行动**：把复杂现实拆成用户可承受的下一小步。
- 面向更边缘、更难触达的人群建立知识与服务能力；
- 与社工、律师、妇联、NGO 等线下资源形成互补，而非竞争。

---

## 6. Current Stage｜当前阶段

目前已完成第一轮专家工作坊与产品测试。

我们获得的主要结论是：

- 一线专家认可“小安”作为第一响应层的价值；
- 现有 generic LLM 在情绪支持上表现不差；
- 当前版本的问题主要来自过重的 flow 和不成熟的知识库；
- 下一阶段重点不是继续堆 prompt，而是重构对话架构与知识体系。

---

## 7. Next Step｜下一步

### 产品与对话架构

- 从“多分支硬路由”重构为“安全强控 + 普通对话弱控”；
- 每轮对话第一步前加入用户信息脱敏，回复用户前进行content safety moderation
- 合并普通场景中的法律、情绪、行动分支，由统一 Main Support Agent 承接混合需求；
- 建立评测集，对比 generic LLM、薄 flow 版本、soft RAG 版本。

### 知识库
- 保留 `knowledge/source/` 原始文档作为 source of truth；
- 按 `AGENTS.md` 的 llm-wiki 规则维护 `knowledge/wiki/`，从核心材料中抽取法律原子单元，构建反家暴相关的 legal mechanism layer；
- 用 `knowledge/index.md` 做内容目录，用 `knowledge/log.md` 记录 ingest / query / lint 历史；
- 同时建立 `knowledge/wiki/scenario-capsules/` 作为 product layer，并在进入生产前进行专家审核。
